# 0008. ООП-first для бизнес-кода, функции для FastAPI роутов

Дата: 2026-04-29
Статус: принято

## Контекст

Стиль написания кода влияет на читаемость, тестируемость и долгосрочную
сопровождаемость. В Python-сообществе уживаются два подхода:

- **ООП-first**: классы для всего, что несёт состояние или поведение.
- **Функциональный/процедурный**: свободные функции, минимум классов.

В проекте есть бизнес-сущности (Product, Order), сценарии
(CreateProduct, UpdateProduct), репозитории — всё это естественно
ложится на классы. Одновременно FastAPI имеет жёсткую идиоматику:
роуты — это функции, и борьба с этим даёт хрупкий код, плохо
читаемый сообществом.

## Решение

Внутри проекта действует правило **ООП-first для бизнес-кода с
исключениями для тонких фреймворочных слоёв**:

- Бизнес-код пишется классами: Entity, Value Object, UseCase,
  Repository, DomainService.
- Свободные функции допустимы для:
  - чистых хелперов без состояния (формат, парсинг);
  - DI-функций FastAPI (`Depends`);
  - pytest-фикстур.
- FastAPI роуты — функции (никаких class-based views, никаких
  сторонних `cbv`-декораторов). Роут — тонкая функция, которая
  принимает UseCase через `Depends`, парсит запрос, вызывает UseCase,
  возвращает ответ.

Граница ООП и функционального стиля совпадает с границей слоёв:
`presentation/` — допустимо функциями; `domain/`, `application/`,
`infrastructure/` — классами.

## Последствия

### Положительные

- Согласованность стиля внутри слоя: домен и application выглядят
  одинаково во всех модулях.
- Единый паттерн `UseCase = класс с методом execute` — легко искать,
  тестировать, ревьювить.
- Никакого переизобретения паттернов FastAPI — роуты выглядят как в
  документации фреймворка.

### Отрицательные / компромиссы

- Граница «когда класс, когда функция» иногда требует осознанного
  решения. Правило «функции только в presentation и для хелперов»
  снимает большую часть случаев.

## Альтернативы

### Полностью функциональный стиль

UseCase как функция, repository как набор функций. Отказались: при
росте кода функции с большим числом параметров становятся
неподдерживаемыми; тестирование с подменой зависимостей через monkeypatch
менее очевидно, чем DI через конструктор.

### Полный ООП включая роуты (class-based views)

Через сторонние библиотеки (`fastapi-utils`, `fastapi-class`).
Отказались: уход от идиоматики FastAPI — это долг, который выплачивается
в каждом будущем обновлении фреймворка.
